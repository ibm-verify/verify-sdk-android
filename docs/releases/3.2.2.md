# IBM Verify SDK for Android - Release 3.2.2

## Overview

Version 3.2.2 introduces QR code-based passwordless login support for cloud authenticators, along with improvements to network logging and MFA service handling. This release enhances the authentication experience by enabling users to approve login requests from their mobile devices without entering passwords.

## 🚨 Breaking Changes

None.

## ✨ New Features

### 1. QR Code Passwordless Login

**New:** [`CloudAuthenticatorService.login()`](../../sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/api/CloudAuthenticatorService.kt:804) method for QR code-based authentication flows.

#### Overview

Enables passwordless login where users scan a QR code and approve the login from their mobile device, eliminating the need to enter passwords.

#### Usage Example

```kotlin
val service = CloudAuthenticatorService(
    _accessToken = authenticator.token.accessToken,
    _refreshUri = authenticator.refreshUri,
    _transactionUri = authenticator.transactionUri,
    _authenticatorId = authenticator.id,
    httpClient = NetworkHelper.getInstance
)

// After scanning QR code and extracting the login code
service.login(
    qrLoginEndpoint = "https://sdk.verify.ibm.com/v1.0/authenticators/login",
    code = "abc123xyz"
).onSuccess {
    println("Login successful")
}.onFailure { error ->
    println("Login failed: ${error.message}")
}
```

#### Method Signature

```kotlin
suspend fun login(
    qrLoginEndpoint: String,
    code: String
): Result<Unit>
```

#### Parameters

- **`qrLoginEndpoint`**: The endpoint URL to submit the login request to
- **`code`**: The login session identifier (LSI) code extracted from the QR code

#### Returns

- **Success:** `Unit` indicating the login was approved
- **Failure:** Exception indicating why the login failed
  - `AuthorizationException` for 401 authentication failures
  - `MFAServiceException.General` for other errors

#### Implementation Details

- Creates JSON payload: `{"code": "<lsi_value>"}`
- Makes POST request with bearer token authentication
- Handles error responses with structured exception types
- Based on v2 SDK implementation in `AuthenticatorContext.loginCloud()`

## 🔧 Improvements

### Core SDK

1. **Network Logging Level Adjustment**
   - Changed default [`NetworkHelper.logLevel`](../../sdk/core/src/main/java/com/ibm/security/verifysdk/core/helper/NetworkHelper.kt:43) from `LogLevel.ALL` to `LogLevel.HEADERS`
   - Reduces verbose logging in production while maintaining essential debugging information
   - Headers-only logging provides sufficient context without exposing sensitive request/response bodies
   - Improves performance by reducing log output overhead

### MFA SDK

1. **Simplified Login Response Handling**
   - Removed unnecessary JSON deserialization in [`MFAServiceDescriptor.login()`](../../sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/MFAServiceDescriptor.kt:227)
   - Changed from `decoder.decodeFromString<Unit>(response.bodyAsText())` to direct `Unit` return
   - Improves performance by avoiding unnecessary parsing of empty/unit responses
   - Maintains identical functionality with cleaner implementation

2. **Enhanced QR Code Login Support**
   - New `login()` method in [`CloudAuthenticatorService`](../../sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/api/CloudAuthenticatorService.kt:805)
   - Comprehensive error handling with structured exceptions
   - Detailed logging for debugging authentication flows
   - Proper cancellation handling for coroutine-based operations

#### Using QR Code Login (New Feature)

If you want to implement QR code-based passwordless login:

1. **Scan QR Code**: Use your QR code scanning implementation to extract the login code
2. **Call Login Method**: Use the new `login()` method on `CloudAuthenticatorService`
3. **Handle Response**: Process success/failure results appropriately

```kotlin
// Example integration
val loginCode = extractCodeFromQrScan(qrCodeData)
val endpoint = extractEndpointFromQrScan(qrCodeData)

service.login(
    qrLoginEndpoint = endpoint,
    code = loginCode
).onSuccess {
    // Navigate to success screen
}.onFailure { error ->
    // Show error message to user
}
```

#### Network Logging Changes (Automatic)

The default network log level has changed from `ALL` to `HEADERS`. If you need full request/response body logging for debugging:

```kotlin
// Restore verbose logging if needed
NetworkHelper.logLevel = LogLevel.ALL
```
