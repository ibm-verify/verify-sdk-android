# IBM Verify SDK for Android - Release 3.2.5

## Overview

Version 3.2.5 is a refinement release that enhances the SSL certificate bypass feature introduced in 3.2.4, improving its documentation, API consistency, and test coverage. This release focuses on making the SSL bypass mechanism more intuitive and better integrated into the registration flow, while maintaining the same security model.

The key improvements center around:
- Simplified HTTP client parameter handling in registration methods
- Enhanced documentation and KDoc comments throughout the SSL bypass implementation
- Expanded test coverage for SSL bypass scenarios
- Better separation of concerns between cloud and on-premise registration providers

## Breaking Changes

None.

This is a backward-compatible patch release. All existing code using 3.2.4 will continue to work without modification.

## Improvements

### 1. Simplified HTTP Client Parameter Handling

**Enhanced:** Registration method signatures across [`MFARegistrationDescriptor`](../../sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/MFARegistrationDescriptor.kt), [`CloudRegistrationProvider`](../../sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/api/CloudRegistrationProvider.kt), and [`OnPremiseRegistrationProvider`](../../sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/api/OnPremiseRegistrationProvider.kt)

#### Overview

All registration enrollment methods now use nullable `HttpClient?` parameters with `null` as the default, making the API more intuitive and consistent with Kotlin conventions.

#### Changes

**Before (3.2.4):**
```kotlin
suspend fun enrollBiometric(httpClient: HttpClient = NetworkHelper.getInstance)
suspend fun enrollUserPresence(httpClient: HttpClient = NetworkHelper.getInstance)
suspend fun enrollOneTimePasscode(httpClient: HttpClient = NetworkHelper.getInstance): OTPAuthenticator
suspend fun finalize(httpClient: HttpClient = NetworkHelper.getInstance): Result<MFAAuthenticatorDescriptor>
```

**After (3.2.5):**
```kotlin
suspend fun enrollBiometric(httpClient: HttpClient? = null)
suspend fun enrollUserPresence(httpClient: HttpClient? = null)
suspend fun enrollOneTimePasscode(httpClient: HttpClient? = null): OTPAuthenticator
suspend fun finalize(httpClient: HttpClient? = null): Result<MFAAuthenticatorDescriptor>
```

#### Benefits

- **Clearer Intent**: `null` explicitly means "use the default behavior"
- **Provider-Specific Defaults**: Each provider can implement its own default client selection logic
- **Better Testing**: Easier to distinguish between "use default" and "use custom client" in tests
- **Consistent Pattern**: Aligns with Kotlin's nullable parameter conventions

#### Implementation Details

**Cloud Provider:**
```kotlin
override suspend fun enrollBiometric(httpClient: HttpClient?) {
    val activeHttpClient = httpClient ?: NetworkHelper.getInstance
    // Always uses secure client
}
```

**OnPremise Provider:**
```kotlin
override suspend fun enrollBiometric(httpClient: HttpClient?) {
    val activeHttpClient = httpClient ?: registrationHttpClient
    // Uses client configured during initiate() (may include SSL bypass)
}
```

### 5. Code Quality Improvements

**Enhanced:** Various code quality improvements throughout the codebase

#### Changes

1. **Consistent Logging**
   - Replaced string concatenation with parameterized logging in `OnPremiseRegistrationProvider`
   - Example: `log.debug("  {}: {}", key, value)` instead of `log.debug("  $key: $value")`
   - Improves performance by avoiding string allocation when logging is disabled

2. **Null Safety**
   - Made `metadataService` nullable in `DetailsData` to handle edge cases
   - Added safe navigation: `metadataService?.serviceName ?: enrollmentEndpoint.host`
   - Prevents potential NullPointerException in malformed server responses

3. **Import Cleanup**
   - Removed unused imports in `OnPremiseRegistrationProvider`
   - Improved code organization and readability

4. **Documentation Consistency**
   - Standardized KDoc formatting across all modified files
   - Consistent use of code blocks, parameter descriptions, and cross-references

### Migration Notes

No code changes are required when upgrading from 3.2.4 to 3.2.5. The API changes are backward-compatible:

- Existing code using `enrollBiometric()` without parameters continues to work
- Existing code passing `NetworkHelper.getInstance` explicitly continues to work
- The nullable parameter change is transparent to existing callers

However, you may want to simplify your code by removing explicit `NetworkHelper.getInstance` parameters:

**Before:**
```kotlin
provider.enrollBiometric(httpClient = NetworkHelper.getInstance)
provider.finalize(httpClient = NetworkHelper.getInstance)
```

**After (simpler):**
```kotlin
provider.enrollBiometric()
provider.finalize()
```
