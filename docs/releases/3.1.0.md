# IBM Verify SDK for Android - Release 3.1.0

## Overview

Version 3.1.0 represents a significant refactoring release focused on improving API consistency, exception handling, and biometric authentication support. This release includes several **breaking changes** that require code modifications when upgrading from 3.0.x versions.

## 🚨 Breaking Changes

### 1. Digital Credentials (DC) SDK Removed

**Impact: HIGH**

The Digital Credentials SDK and demo app have been completely removed from this repository.

- **Removed Components:**
  - `sdk/dc` module
  - `examples/dc_demo` application
  - All DC-related APIs, models, and serializers

- **Migration Path:**
  The DC functionality has been migrated to a separate repository: [IBM Verify DC Wallet SDK for Android](https://github.com/ibm-verify/verify-dc-wallet-sdk-android)

- **Action Required:**
  If you are using the DC SDK, you must:
  1. Remove the `verify-sdk-dc` dependency from your `build.gradle.kts`
  2. Migrate to the new DC Wallet SDK repository
  3. Update imports from `com.ibm.security.verifysdk.dc.*` to the new package structure

### 2. MFA SDK - Error to Exception Migration

**Impact: HIGH**

All error classes have been replaced with exception classes for better error handling.

#### Replaced Classes:

| Old Class (3.0.x) | New Class (3.1.0) | Location |
|-------------------|-------------------|----------|
| `MFARegistrationError` | [`MFARegistrationException`](sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/MFARegistrationException.kt) | MFA SDK |
| `MFAServiceError` | [`MFAServiceException`](sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/MFAServiceException.kt) | MFA SDK |
| `HashAlgorithmError` | [`HashAlgorithmException`](sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/HashAlgorithmException.kt) | MFA SDK |

#### Migration Example:

**Before (3.0.x):**
```kotlin
when (result) {
    is Result.Failure -> {
        when (result.error) {
            is MFARegistrationError.NoEnrollableFactors -> {
                // Handle error
            }
        }
    }
}
```

**After (3.1.0):**
```kotlin
try {
    // MFA registration code
} catch (e: MFARegistrationException.NoEnrollableFactors) {
    // Handle exception
} catch (e: MFARegistrationException) {
    // Handle other registration exceptions
}
```

#### New Exception Types:

**[`MFARegistrationException`](sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/MFARegistrationException.kt):**
- `FailedToParse` - JSON parsing failure
- `InvalidFormat` - Invalid JSON format for MFARegistrationDescriptor
- `NoEnrollableFactors` - No factors available for enrollment
- `EnrollmentFailed` - Factor enrollment failed
- `DataInitializationFailed` - Initialization failed
- `MissingAuthenticatorIdentifier` - Missing authenticator ID from OAuth token
- `InvalidRegistrationData` - Invalid registration data
- `SignatureMethodNotEnabled` - Signature method not enabled
- `InvalidAlgorithm` - Invalid algorithm provided
- `General` - General registration error

**[`MFAServiceException`](sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/MFAServiceException.kt):**
- `InvalidSigningHash` - Invalid signing hash algorithm
- `InvalidPendingTransaction` - No pending transaction available
- `SerializationFailed` - Serialization conversion failed
- `InvalidDataResponse` - Invalid response data
- `DecodingFailed` - JSON decoding failed
- `UnableToCreateTransaction` - Unable to create pending transaction
- `General` - General MFA error

### 3. MFA SDK - Biometric Factor Consolidation

**Impact: MEDIUM**

Face and fingerprint factors have been unified into a single biometric factor type.

#### Changes:

| Old (3.0.x) | New (3.1.0) |
|-------------|-------------|
| `FactorType.Face` | `FactorType.Biometric` |
| `FactorType.Fingerprint` | `FactorType.Biometric` |
| `FaceFactorInfo` | [`BiometricFactorInfo`](sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/BiometricFactorInfo.kt) |
| `FingerprintFactorInfo` | [`BiometricFactorInfo`](sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/BiometricFactorInfo.kt) |

#### Migration Example:

**Before (3.0.x):**
```kotlin
when (factor) {
    is FactorType.Face -> handleFace(factor.value)
    is FactorType.Fingerprint -> handleFingerprint(factor.value)
}
```

**After (3.1.0):**
```kotlin
when (factor) {
    is FactorType.Biometric -> handleBiometric(factor.value)
}
```

### 4. MFA SDK - Authenticator Descriptor Changes

**Impact: MEDIUM**

The [`MFAAuthenticatorDescriptor`](sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/MFAAuthenticatorDescriptor.kt) interface has been restructured.

#### Removed:
- `allowedFactors: List<FactorType>` property

#### Added:
- `biometric: BiometricFactorInfo?` - Biometric factor information
- `userPresence: UserPresenceFactorInfo?` - User presence factor information

#### Migration Example:

**Before (3.0.x):**
```kotlin
val factors = authenticator.allowedFactors
factors.forEach { factor ->
    when (factor) {
        is FactorType.Face -> // Handle face
        is FactorType.Fingerprint -> // Handle fingerprint
    }
}
```

**After (3.1.0):**
```kotlin
authenticator.biometric?.let { biometric ->
    // Handle biometric factor
}

authenticator.userPresence?.let { userPresence ->
    // Handle user presence factor
}
```

### 5. MFA SDK - Registration Method Changes

**Impact: MEDIUM**

The [`MFARegistrationDescriptor`](sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/MFARegistrationDescriptor.kt) now provides separate methods for biometric and user presence enrollment.

#### New Methods:
- `enrollBiometric()` - Enroll biometric factor
- `enrollUserPresence()` - Enroll user presence factor

#### Migration Example:

**Before (3.0.x):**
```kotlin
// Generic enrollment based on factor type
registrationDescriptor.enroll(factorType)
```

**After (3.1.0):**
```kotlin
// Specific enrollment methods
registrationDescriptor.enrollBiometric()
registrationDescriptor.enrollUserPresence()
```

### 6. Core SDK - Exception Handling Enhancement

**Impact: LOW to MEDIUM**

[`AuthenticationException`](sdk/core/src/main/java/com/ibm/security/verifysdk/core/AuthenticationException.kt) and [`AuthorizationException`](sdk/core/src/main/java/com/ibm/security/verifysdk/core/AuthorizationException.kt) now include a `cause` parameter.

#### Changes:

**Before (3.0.x):**
```kotlin
class AuthenticationException(
    val code: HttpStatusCode,
    id: String,
    description: String
)
```

**After (3.1.0):**
```kotlin
class AuthenticationException(
    val code: HttpStatusCode,
    id: String,
    description: String,
    cause: Throwable? = null
)
```

#### Benefits:
- Better error chaining and debugging
- Preserves underlying exception information
- Improved stack traces

### 7. Core SDK - VerifySdkException Refactoring

**Impact: LOW**

[`VerifySdkException`](sdk/core/src/main/java/com/ibm/security/verifysdk/core/VerifySdkException.kt) now uses a structured `Error` data class.

#### Changes:

**Before (3.0.x):**
```kotlin
open class VerifySdkException(
    val errorId: String,
    message: String
) : Exception(message)
```

**After (3.1.0):**
```kotlin
open class VerifySdkException(
    val error: Error,
    override val cause: Throwable? = null
) : Exception(error.description, cause)
```

Where `Error` is:
```kotlin
data class Error(val id: String, val description: String)
```

#### Migration Example:

**Before (3.0.x):**
```kotlin
val errorId = exception.errorId
val message = exception.message
```

**After (3.1.0):**
```kotlin
val errorId = exception.error.id
val description = exception.error.description
val cause = exception.cause
```

### 8. MFA SDK - EnrollableFactor Interface Change

**Impact: LOW**

The internal [`EnrollableFactor`](sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/EnrollableFactor.kt) interface now includes an `enabled` property.

#### Added:
```kotlin
internal interface EnrollableFactor {
    val uri: URL
    val type: EnrollableType
    val enabled: Boolean  // NEW
}
```

This change primarily affects internal SDK implementation and custom factor implementations.

## ✨ New Features

### 1. Enhanced HOTP Passcode Generation

The [`HOTPFactorInfo`](sdk/mfa/src/main/java/com/ibm/security/verifysdk/mfa/HOTPFactorInfo.kt) class now provides better control over counter management.

#### New Method:
```kotlin
fun generatePasscode(incrementCounter: Boolean): String
```

#### Usage:
```kotlin
// Generate passcode and increment counter
val passcode1 = hotpFactor.generatePasscode()

// Preview current passcode without incrementing counter
val passcode2 = hotpFactor.generatePasscode(incrementCounter = false)

// Generate passcode with explicit counter control
val passcode3 = hotpFactor.generatePasscode(incrementCounter = true)
```

#### Benefits:
- Preview passcodes without consuming them
- Better control over counter state
- Useful for testing and debugging

### 2. Improved TOTP Counter Calculation

TOTP counter calculation has been optimized by removing unnecessary multiplication.

**Before (3.0.x):**
```kotlin
val counter = (Date().time / 1000) / (period * 1000)
```

**After (3.1.0):**
```kotlin
val counter = (Date().time / 1000) / period
```

This ensures more accurate time-based OTP generation.

### 3. Enhanced BiometricAuthenticationException

[`BiometricAuthenticationException`](sdk/fido2/src/main/java/com/ibm/security/verifysdk/fido2/BiometricAuthenticationException.kt) now extends [`VerifySdkException`](sdk/core/src/main/java/com/ibm/security/verifysdk/core/VerifySdkException.kt) for consistent error handling across the SDK.

#### Benefits:
- Unified exception hierarchy
- Better error information structure
- Consistent error handling patterns

## 🔧 Improvements

### Core SDK

1. **Enhanced Base32 Decoding**
   - Now RFC 4648 compliant
   - Case-insensitive handling
   - Robust optional padding support

2. **URL Manipulation**
   - `URL.replace()` renamed to `URL.replaceInPath()` for clarity
   - More flexible URI handling

3. **Thread-Safe HttpClient**
   - `client` property now volatile
   - Double-checked locking for safe initialization
   - New `invalidateClient()` method replaces `reinitializeClient()`
   - Proper client cleanup on configuration changes

4. **KeystoreHelper Logging**
   - Added warning logs for key deletion operations
   - Improved traceability for key management

### MFA SDK

1. **Simplified Factor Management**
   - Removed generic `allowedFactors` list
   - Direct access to specific factor types
   - Clearer API surface

2. **Better Enrollment Control**
   - Separate methods for different enrollment types
   - More explicit enrollment flow
   - Reduced complexity

3. **Enhanced Error Information**
   - Structured error data with `Error` class
   - Cause chaining for better debugging
   - Consistent error handling patterns

## 🔄 Migration Guide

### Step 1: Update Dependencies

Update your `build.gradle.kts`:

```kotlin
dependencies {
    // Update to 3.1.0
    implementation("com.github.ibm-verify.verify-sdk-android:verify-sdk-authentication:3.1.0")
    implementation("com.github.ibm-verify.verify-sdk-android:verify-sdk-mfa:3.1.0")
    implementation("com.github.ibm-verify.verify-sdk-android:verify-sdk-core:3.1.0")
    implementation("com.github.ibm-verify.verify-sdk-android:verify-sdk-fido2:3.1.0")
    implementation("com.github.ibm-verify.verify-sdk-android:verify-sdk-adaptive:3.1.0")
    
    // Remove DC SDK if present
    // implementation("com.github.ibm-verify.verify-sdk-android:verify-sdk-dc:3.0.x") // REMOVE
}
```

### Step 2: Update Error Handling

Replace all error handling with exception handling:

```kotlin
// OLD - Error-based handling
when (result) {
    is Result.Failure -> {
        when (result.error) {
            is MFARegistrationError -> // Handle
        }
    }
}

// NEW - Exception-based handling
try {
    // MFA operations
} catch (e: MFARegistrationException) {
    when (e) {
        is MFARegistrationException.NoEnrollableFactors -> // Handle
        is MFARegistrationException.EnrollmentFailed -> // Handle
        else -> // Handle other cases
    }
}
```

### Step 3: Update Factor Type Handling

Replace face/fingerprint with biometric:

```kotlin
// OLD
when (factor) {
    is FactorType.Face -> handleFace()
    is FactorType.Fingerprint -> handleFingerprint()
}

// NEW
when (factor) {
    is FactorType.Biometric -> handleBiometric()
}
```

### Step 4: Update Authenticator Access

Replace `allowedFactors` with direct property access:

```kotlin
// OLD
authenticator.allowedFactors.forEach { factor ->
    // Process factor
}

// NEW
authenticator.biometric?.let { handleBiometric(it) }
authenticator.userPresence?.let { handleUserPresence(it) }
```

### Step 5: Update Exception Property Access

Update exception property access:

```kotlin
// OLD
val errorId = exception.errorId
val message = exception.message

// NEW
val errorId = exception.error.id
val description = exception.error.description
val cause = exception.cause
```

### Step 6: Migrate DC SDK (If Applicable)

If using DC SDK:
1. Remove `verify-sdk-dc` dependency
2. Add dependency on [IBM Verify DC Wallet SDK](https://github.com/ibm-verify/verify-dc-wallet-sdk-android)
3. Update imports and API calls according to new SDK documentation
